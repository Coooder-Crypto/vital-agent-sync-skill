# Changelog

## 0.5.2 - 2026-07-19

- 固定安装已发布的 `vitalmcp@0.5.2`。
- 固定安装时使用的 Node 路径和原生模块 ABI，避免 WorkBuddy Node 与系统 Node 混用。
- 在 WorkBuddy 沙箱无法激活 launchd 时暂停流程，只交付运行时生成的 Terminal 恢复命令。
- 要求用户在 WorkBuddy 中审批并重载 MCP，且原生 `vital_agent_status` 验证成功后才能继续配对。
- 禁止修改 MCP 审批存储、递归清理 quarantine 属性或绕过 MCP 直接读取 SQLite。

## 0.5.1 - 2026-07-18

- 固定安装已发布的 `vitalmcp@0.5.1`。
- 在复用本地接收端前验证运行时、协议和数据库身份，遇到旧版或未知进程时安全停止。
- 服务管理器启动失败时保留原始错误，禁止使用自制后台进程或修改 shell 配置绕过正式流程。
- 禁止未经确认的数据迁移、SQLite 直读和原生 MCP 之外的健康数据访问。
- 在首次健康数据读取前要求明确的模型隐私提示和用户确认。

## 0.5.0 - 2026-07-15

- 首次公开发布 WorkBuddy Skill。
- 固定安装 `vitalmcp@0.5.0`。
- 支持先审阅计划、再确认执行的一句话安装流程。
- 支持本地服务、WorkBuddy MCP 配置、局域网二维码配对与首次同步验证。
- 增加健康数据最小化、模型提供商披露和敏感凭据保护规则。
